Hướng dẫn tạo Blog với Hugo và Deploy lên GitHub Pages#

Hướng dẫn chi tiết từng bước để tạo một blog với Hugo và deploy lên GitHub Pages hoàn toàn miễn phí. Các bước được thực hiện trên Windows.

1. Cài đặt Hugo#

Cách 1: Sử dụng Chocolatey (Khuyến nghị)#

Mở PowerShell với quyền Administrator và chạy:

1# Cài đặt Chocolatey nếu chưa có
2Set-ExecutionPolicy Bypass -Scope Process -Force; [System.Net.ServicePointManager]::SecurityProtocol = [System.Net.ServicePointManager]::SecurityProtocol -bor 3072; iex ((New-Object System.Net.WebClient).DownloadString('https://community.chocolatey.org/install.ps1'))
3
4# Cài đặt Hugo
5choco install hugo-extended -y

Cách 2: Download trực tiếp từ GitHub#

  1. Truy cập Hugo Releases
  2. Download file hugo_extended_x.x.x_windows-amd64.zip
  3. Giải nén và copy file hugo.exe vào thư mục C:\Hugo\bin
  4. Thêm C:\Hugo\bin vào PATH environment variable

Kiểm tra cài đặt#

Mở Command Prompt hoặc PowerShell và chạy:

1hugo version

2. Tạo Hugo Site mới#

Mở Command Prompt hoặc PowerShell, navigate đến thư mục để tạo blog:

1# Tạo site mới
2hugo new site my-blog
3
4# Di chuyển vào thư mục
5cd my-blog
6
7# Khởi tạo Git repository
8git init

Cấu trúc thư mục sẽ như sau:

1my-blog/
2├── archetypes/
3├── assets/
4├── content/
5├── data/
6├── layouts/
7├── static/
8├── themes/
9└── hugo.toml

3. Chọn và cài đặt Theme#

Tìm Theme#

Truy cập Hugo Themes để chọn theme phù hợp.

Cài đặt Theme (Ví dụ: PaperMod)#

1# Thêm theme như một Git submodule
2git submodule add --depth=1 https://github.com/adityatelange/hugo-PaperMod.git themes/PaperMod

Cấu hình Theme cơ bản#

Mở file hugo.toml và cấu hình:

 1baseURL = 'https://your-username.github.io'
 2languageCode = 'vi-VN'
 3title = 'My Blog'
 4theme = 'PaperMod'
 5
 6[params]
 7    author = "Your Name"
 8    description = "My personal blog"
 9    
10    [params.homeInfoParams]
11        Title = "Chào mừng đến với blog của tôi"
12        Content = "Nơi chia sẻ kiến thức và kinh nghiệm"
13
14[[menu.main]]
15    name = "Home"
16    url = "/"
17    weight = 10
18
19[[menu.main]]
20    name = "Posts"
21    url = "/posts/"
22    weight = 20
23
24[[menu.main]]
25    name = "About"
26    url = "/about/"
27    weight = 30

4. Tạo các trang cơ bản#

Khi mới khởi tạo Hugo site, bạn sẽ không có các trang cơ bản như _index, about, archives. Đây là cách tạo chúng:

Tạo trang chủ (_index.md)#

1hugo new _index.md

Hoặc tạo thủ công file content/_index.md:

1---
2title: "Trang chủ"
3date: 2025-01-16
4draft: false
5---
6
7# Chào mừng đến với blog của tôi!
8
9Đây là trang chủ của blog Hugo. Nơi tôi chia sẻ những kiến thức và kinh nghiệm của mình.

Tạo trang Giới thiệu (About)#

1hugo new about.md

Hoặc tạo file content/about.md:

 1---
 2title: "Giới thiệu"
 3date: 2025-01-16
 4draft: false
 5ShowToc: false
 6ShowReadingTime: false
 7---
 8
 9# Xin chào! Tôi là [Tên của bạn]
10
11## Về tôi
12Giới thiệu bản thân, sở thích, kinh nghiệm...
13
14## Liên hệ
15- Email: your-email@example.com
16- GitHub: [github.com/username](https://github.com/username)

Tạo trang Lưu trữ (Archives)#

1hugo new archives.md

Hoặc tạo file content/archives.md:

1---
2title: "Lưu trữ"
3layout: "archives"
4url: "/archives/"
5summary: archives
6---

Tạo trang Contact (Liên hệ) - Tùy chọn#

1hugo new contact.md

Hoặc tạo file content/contact.md:

 1---
 2title: "Liên hệ"
 3date: 2025-01-16
 4draft: false
 5---
 6
 7# Liên hệ với tôi
 8
 9## Thông tin liên hệ
10- **Email**: your-email@example.com
11- **GitHub**: [github.com/username](https://github.com/username)
12- **LinkedIn**: [linkedin.com/in/username](https://linkedin.com/in/username)
13
14## Gửi tin nhắn
15Bạn có thể gửi email trực tiếp hoặc liên hệ qua các kênh mạng xã hội trên.

Tạo trang Search (Tìm kiếm) - Cho themes hỗ trợ#

1hugo new search.md

Hoặc tạo file content/search.md:

1---
2title: "Tìm kiếm"
3layout: "search"
4---

5. Tạo bài viết đầu tiên#

1# Tạo bài viết mới
2hugo new posts/my-first-post.md

Chỉnh sửa file content/posts/my-first-post.md:

 1---
 2title: "Bài viết đầu tiên của tôi"
 3date: 2024-12-19T10:00:00+07:00
 4draft: false
 5description: "Đây là bài viết đầu tiên trên blog Hugo của tôi"
 6tags: ["Hugo", "Blog"]
 7---
 8
 9# Chào mừng đến với blog của tôi!
10
11Đây là bài viết đầu tiên được viết bằng Hugo. Hugo là một static site generator rất nhanh và mạnh mẽ.
12
13## Tại sao chọn Hugo?
14
15- **Nhanh**: Build site trong vài giây
16- **Linh hoạt**: Hỗ trợ nhiều theme và customization
17- **Miễn phí**: Open source và deploy miễn phí

Preview Local#

1hugo server -D

Truy cập http://localhost:1313 để xem blog.

7. Cấu hình GitHub Repository#

Tạo Repository trên GitHub#

  1. Đăng nhập GitHub
  2. Click “New repository”
  3. Đặt tên: your-username.github.io (thay your-username bằng username GitHub)
  4. Check “Public”
  5. Click “Create repository”

Kết nối Local với GitHub#

 1# Thêm remote origin
 2git remote add origin https://github.com/your-username/your-username.github.io.git
 3
 4# Add và commit files
 5git add .
 6git commit -m "Initial commit: Hugo site setup"
 7
 8# Push lên GitHub
 9git branch -M main
10git push -u origin main

8. Deploy lên GitHub Pages#

Cách 1: Manual Deploy#

1# Build site
2hugo
3
4# Copy public folder content to root (cho GitHub Pages)
5# Hoặc sử dụng branch gh-pages

Cách 2: Sử dụng GitHub Pages Settings#

  1. Vào repository trên GitHub
  2. Click “Settings” tab
  3. Scroll xuống “Pages” section
  4. Chọn source: “Deploy from a branch”
  5. Chọn branch: main và folder: / (root)

9. Tự động hóa với GitHub Actions#

Tạo file .github/workflows/hugo.yml:

 1name: Deploy Hugo site to Pages
 2
 3on:
 4  push:
 5    branches:
 6      - main
 7  workflow_dispatch:
 8
 9permissions:
10  contents: read
11  pages: write
12  id-token: write
13
14concurrency:
15  group: "pages"
16  cancel-in-progress: false
17
18defaults:
19  run:
20    shell: bash
21
22jobs:
23  build:
24    runs-on: ubuntu-latest
25    env:
26      HUGO_VERSION: 0.120.4
27    steps:
28      - name: Install Hugo CLI
29        run: |
30          wget -O ${{ runner.temp }}/hugo.deb https://github.com/gohugoio/hugo/releases/download/v${HUGO_VERSION}/hugo_extended_${HUGO_VERSION}_linux-amd64.deb \
31          && sudo dpkg -i ${{ runner.temp }}/hugo.deb          
32      
33      - name: Install Dart Sass
34        run: sudo snap install dart-sass
35        
36      - name: Checkout
37        uses: actions/checkout@v4
38        with:
39          submodules: recursive
40          fetch-depth: 0
41          
42      - name: Setup Pages
43        id: pages
44        uses: actions/configure-pages@v3
45        
46      - name: Install Node.js dependencies
47        run: "[[ -f package-lock.json || -f npm-shrinkwrap.json ]] && npm ci || true"
48        
49      - name: Build with Hugo
50        env:
51          HUGO_ENVIRONMENT: production
52          HUGO_ENV: production
53        run: |
54          hugo \
55            --gc \
56            --minify \
57            --baseURL "${{ steps.pages.outputs.base_url }}/"          
58            
59      - name: Upload artifact
60        uses: actions/upload-pages-artifact@v2
61        with:
62          path: ./public
63
64  deploy:
65    environment:
66      name: github-pages
67      url: ${{ steps.deployment.outputs.page_url }}
68    runs-on: ubuntu-latest
69    needs: build
70    steps:
71      - name: Deploy to GitHub Pages
72        id: deployment
73        uses: actions/deploy-pages@v2

Commit và Push#

1git add .
2git commit -m "Add GitHub Actions workflow for Hugo deployment"
3git push

Kết quả#

Sau khi setup thành công, blog sẽ có sẵn tại https://your-username.github.io

Lưu ý quan trọng#

  1. Thời gian deploy: GitHub Pages có thể mất 5-10 phút để reflect changes
  2. Custom domain: Có thể cấu hình custom domain trong Settings > Pages
  3. HTTPS: GitHub Pages tự động cung cấp HTTPS
  4. Cập nhật content: Mỗi khi push code mới, blog sẽ tự động rebuild
  5. Theme customization: Nên tạo custom CSS trong assets/css/ thay vì sửa trực tiếp theme
  6. Backup: Luôn backup cấu hình và content của bạn

Kết luận#

Sau khi hoàn thành các bước trên, blog Hugo đã được tạo và deploy thành công lên GitHub Pages! Từ đây, chỉ cần viết bài trong thư mục content/posts/ và push lên GitHub, blog sẽ tự động cập nhật.

Các bước tiếp theo có thể thực hiện:

  • Customize theme theo nhu cầu
  • Thêm Google Analytics
  • Cấu hình SEO
  • Thêm comment system
  • Tạo custom layouts
  • Thêm shortcodes
  • Tối ưu performance

Mẹo hữu ích:

  • Sử dụng hugo server -D --navigateToChanged để tự động reload và navigate
  • Tạo script để tự động tạo bài viết mới
  • Sử dụng Hugo modules thay vì git submodules cho themes
  • Backup regular với git
